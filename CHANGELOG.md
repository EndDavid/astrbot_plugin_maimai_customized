# 更新日志

## 1.2.2

- 优化“吃分推荐”推送逻辑：从按规则排序后的前最多 10 个候选谱面中随机推荐；候选不足 10 首时按实际候选数量抽取，候选为 0 时直接返回无候选提示。

## 1.2.1

- 优化“吃分推荐”输出：移除单独处理中提示，将推荐理由和曲目信息图合并为一条引用回复。
- 新增按用户分装的最近 3 次推荐谱面栈；候选超过 3 首时优先避开最近已推荐谱面。
- 推荐理由移除筛选过程说明，新增 `b50倾向` 标签统计。
- 添加“定拍”标签，优化标签映射。

## 1.2.0

- 优化“吃分推荐”响应流程：立即返回处理中提示，将候选计算和谱面详情图生成移出主事件循环，并限制并发请求，降低短时间触发导致 AstrBot 无响应的风险。
- 调整“吃分推荐”筛选规则：按当前 Rating 推导推荐定数区间，定数范围为 `当前 Rating / 1130` 到 `当前 Rating / 1075`，依次判断 B35/B15 地板、SSS+ 理论单曲 Rating、B50 中已 SSS+ 谱面排除，并优先推荐实际定数更低的谱面，其次推荐“拟合定数 - 实际定数”更低的谱面。
- B35/B15 地板只统计已有正单曲 Rating，忽略分值为 0 的占位成绩。
- 推送更新后的 `static/maimaidx_chart_tags.json` 静态标签库。

## 1.1.3

- 修复 WebUI 已配置 Token 后，公网或 NAT 访问根路径仍容易表现为“拒绝启动 / 403”的问题；现在访问根路径会显示 Token 输入页，接口仍保持 Token 校验。
- 优化了“谱面标签”的获取逻辑。
- 修复了牌子数据接口读取异常的Bug。

## 1.1.2

- WebUI 外部监听必须配置访问 Token；未配置 Token 时仅允许本机访问。
- 插件初始化改为分项降级，曲库、别名或牌子数据加载失败时不阻断命令注册和定时更新。
- 搜索、分数线和猜歌图片改为直接发送 base64 图片组件，减少临时文件残留。
- 插件停止时会清理猜歌后台任务、WebUI 启动任务和谱面标签更新任务。

## 1.1.1

首个发行版本，提供面向 AstrBot 的舞萌 DX 查询与管理能力：歌曲查询、成绩查询、B50 图片、B50 锐评、成绩同步、谱面标签、吃分推荐、猜歌和插件 WebUI。

安装、配置、静态资源准备和完整命令说明请查看 `README.md`。
